The error you're seeing:
7503 — "Application is not registered in our store."
This means Microsoft's calling infrastructure doesn't recognize your app as a Bot. The steps below will fix this.

Prerequisites

Before you begin, make sure you have:

  • An Azure account with an active subscription
  • An existing Azure AD App Registration (the one your backend uses — AZURE_CLIENT_ID)
  • Tenant admin access (needed to grant API permissions)
  • ngrok installed for local development — your current URL: https://351f-115-186-174-60.ngrok-free.app

Step 1 — Create an Azure Bot Resource

1 Go to portal.azure.com

2 In the search bar at the top, type "Azure Bot" and click Azure Bot under Services.

3 Click + Create

4 Fill in the creation form:

FieldValue
Bot handleteams-meeting-assistant-bot (or any unique name)
SubscriptionYour Azure subscription
Resource groupSame resource group as your app registration (or create new)
Data residencyGlobal (or your preferred region)
Pricing tierF0 (Free) — sufficient for development
Type of AppMulti Tenant
Creation typeUse existing app registration
App IDYour AZURE_CLIENT_ID value from .env
App tenant IDYour AZURE_TENANT_ID value from .env
Important: Select "Use existing app registration" — do NOT create a new one. You must use the same AZURE_CLIENT_ID that your backend is already configured with.

5 Click Review + Create, then Create. Wait for deployment to complete.

6 Once deployed, click "Go to resource".

7 In the left sidebar, click Configuration. Set the Messaging endpoint to:

https://351f-115-186-174-60.ngrok-free.app/api/meeting-bot/callback

Then click Apply.

Note: The ngrok URL changes every time you restart ngrok (on the free plan). You'll need to update this messaging endpoint each time. For production, use your real HTTPS domain.

Step 2 — Enable the Microsoft Teams Channel

This is the step that registers your app in the "Bot Framework store" — the exact thing the 7503 error says is missing.

1 In your Azure Bot resource, click Channels in the left sidebar.

2 Under "Available channels", find Microsoft Teams and click on it.

3 On the configuration page:

  • Accept the Terms of Service
  • Under "Calling" tab, check "Enable calling"
  • Set the Webhook URL to: https://351f-115-186-174-60.ngrok-free.app/api/meeting-bot/callback

4 Click Apply / Save.

This is the critical step. Once Teams channel is enabled, your app is registered in the Bot Framework store and the 7503 error will be resolved.

Step 3 — Add Graph API Permissions

Your app needs specific application permissions to join calls and access media.

1 Go to Azure Portal → Azure Active Directory → App registrations

2 Find and click your app (the one matching AZURE_CLIENT_ID).

3 Click API permissions in the left sidebar.

4 Click + Add a permission → Microsoft Graph → Application permissions.

5 Search for and add each of these permissions:

PermissionPurpose
Calls.JoinGroupCall.AllBot can join group calls / meetings
Calls.InitiateGroupCall.AllBot can create outbound calls
Calls.AccessMedia.AllBot can access audio/video streams
OnlineMeetings.Read.AllBot can read meeting info & recordings
OnlineMeetings.ReadWrite.AllBot can manage online meetings

6 Click Add permissions after selecting all five.

Check existing permissions too. You should already have some delegated permissions for user login. Keep those — just make sure the five application permissions above are also added.

Step 4 — Grant Admin Consent

1 Still on the API permissions page, click the "Grant admin consent for [your tenant]" button at the top.

2 Click Yes to confirm.

3 Verify all permissions show a green checkmark under "Status" saying "Granted for [tenant]".

You must be a tenant admin to grant consent. If you're not, ask your IT admin to click this button for you. Without admin consent, the permissions won't work even though they're listed.

1 You already have ngrok running:

Your current public URL: https://351f-115-186-174-60.ngrok-free.app

2 Make sure these three places all use the same ngrok URL:

LocationValue
Azure Bot → Configuration → Messaging endpointhttps://351f-...ngrok-free.app/api/meeting-bot/callback

Step 6 — Update Your .env Configuration

Your backend configuration file already has the botCallbackUrl set. Verify your .env file has these values:

# Azure AD (should already be set)
AZURE_CLIENT_ID=your-client-id-here
AZURE_CLIENT_SECRET=your-client-secret-here
AZURE_TENANT_ID=your-tenant-id-here

# Bot callback URL (update when ngrok URL changes)
BOT_CALLBACK_URL=https://351f-115-186-174-60.ngrok-free.app/api/meeting-bot/callback

Then restart your backend server so it picks up the new config:

npm run start:dev

Step 7 — Test the Bot

1 Start a Teams meeting (or use a scheduled meeting with a join URL).

2 Go to your app's Meeting Bot page (/meeting-bot).

3 Paste the Teams meeting join URL and click "Join Meeting & Record".

4 Check your ngrok terminal — you should see incoming POST requests to /api/meeting-bot/callback.

5 The bot should appear as a participant in the Teams meeting within 10-15 seconds.

If the bot joins successfully, the full pipeline is: Join → Record → (when you click Stop or meeting ends) → Download recording → Whisper transcription → GPT-4o summary. All automatic.

Troubleshooting

Error 7503 still appearing

Wait 5-10 minutes after enabling the Teams channel. Bot registration can take a few minutes to propagate through Microsoft's systems. Then restart your backend and try again.

Error 403 / Forbidden

Admin consent hasn't been granted. Go back to Step 4 and click "Grant admin consent." Every permission should show a green checkmark.

Error 401 / Unauthorized

Your AZURE_CLIENT_SECRET may have expired. Go to your App Registration → Certificates & secrets → create a new client secret → update your .env file.

Bot joins but no recording

The Calls.AccessMedia.All permission requires a Teams policy update in some tenants. Your Teams admin may need to create a policy that allows application-hosted media bots. Run this in PowerShell:

New-CsApplicationAccessPolicy -Identity "MeetingBot-Policy" -AppIds "YOUR_CLIENT_ID" -Description "Allow meeting bot"
Grant-CsApplicationAccessPolicy -PolicyName "MeetingBot-Policy" -Global

Callback not receiving notifications

Check that:

  • ngrok is running and the URL matches what's in Azure Bot configuration
  • Your backend is running on port 3001
  • The /api/meeting-bot/callback endpoint returns 200 OK (test with curl -X POST https://351f-...ngrok-free.app/api/meeting-bot/callback)
